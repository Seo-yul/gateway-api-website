Matches are generated from the conformance reports uploaded by project implementations. Must have determines which controllers match; Nice to have optionally ranks your results. Feature names match the v1.4 conformance tables. Implementations only appear in this page if they pass Core conformance for the resource type. Implementations that submit conformance reports with skipped tests won't appear.

Gateway

All controllers matched by this wizard include core support of Gateway. That includes many core features, including:

  • Specifying listeners on different ports and protocols
  • TLS configuration (protocol dependent)
  • Hostname matching (protocol dependent)
  • Configuring which Routes can be attached, including the ability to accept Routes from other namespaces

For extended Gateway features, first select the Gateway Resource, and then you can choose from the following list of extended (optional) features.

GatewayRequirement

HTTPRoute

Controllers that pass conformance tests for HTTPRoute are guaranteed to support the following features:

  1. Traffic splitting with weighted backends
  2. Safe cross-namespace references with ReferenceGrant
  3. Request header modification
  4. Exact and prefix path matching
  5. Hostname matching
  6. HTTP redirects

For extended HTTPRoute features, first select the HTTPRoute Resource, and then you can choose from the following list of extended (optional) features.

GRPCRoute

All GRPCRoute features are included as part of the core conformance, there are no optional features.

TLSRoute

All TLSRoute features are included as part of the core conformance, there are no optional features.
